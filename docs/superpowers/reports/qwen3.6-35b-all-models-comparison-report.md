## 六、结论与建议

### 6.1 🏆 推荐模型

| 场景 | 推荐模型 | 得分 | 理由 |
|------|---------|:----:|------|
| 🥇 **综合最佳** | havenoammo/qwen3.6-35b-a3b-mtp | 66.05 | 总分第一，性能 95.0 |
| 🥇 **轻量高效** | nvidia/nemotron-3-nano-omni | 61.94 | 仅 3.56 GiB，性能 95.0，接近 35b 中游水平 |
| 🥇 **内存优先** | qwopus-moe-35b-a3b@q4_k_m | 63.70 | 仅 19.71 GiB，得分合理 |
| 🥇 **推理最优** | reasoning-distilled | 63.39 | 推理维度最高分 |
| 🥇 **Agent最优** | uncensored-aggressive | 65.53 | Agent 维度最高分 |
| 🥇 **性价比之选** | qwopus3.6-35b-a3b-v1 | 65.62 | 仅次于冠军，全面均衡 |
| 🥇 **代码最优** | qwen3.5-27b-reasoning-distilled | 63.10 | 代码维度历史最高分 |

### 6.2 关键发现

1. **所有 35b MoE 模型得分集中**：61.97 ~ 66.05，极差仅 4.08pp（仅统计 35b 同类架构）
2. **MTP 版本普遍优于非 MTP**：MTP 版占前 10 名中 6 席
3. **量化影响有限**：oq8、mlx-oq8 等量化版本与原始版得分差距 < 1pp
4. **Abilterated 系列表现偏弱**：huihui 的三个版本排名 13-16，建议优先选择原版
5. **Agent 能力与推理能力呈反相关**：Agent 最高（uncensored 65.53）的模型推理不是最高（61.07）
6. **性能测试差异大**：95 分与 82 分相差 13pp，影响总分约 1.6pp
7. **27b 密集模型代码能力突出**：qwen3.5-27b 以 63.10 登顶代码榜，但 TPS 仅 8.3，推理速度是 MoE 模型的 1/5
8. **coder-next 架构特殊但 Agent 弱**：84.81 GiB 的 512x2.5B MoE 推理不错（62.03），但 Agent 维度仅 36.73 分，明显拖累总分
9. **nemotron 小模型令人惊喜**：3B 参数以 61.94 总分接近 35b 中游水平，性能 95.0 满分，是轻量部署的绝佳选择
